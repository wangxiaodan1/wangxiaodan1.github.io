关于IOS javascript date 的bug

========== =========== =========== ==============

IOS safari 中不能解析的时间格式

1.需解析的时间格式(三种)

2017-03-12 12:00:00
2017-03-12
2017-3-12

2.safari浏览器 解析上面时间格式

（1）.使用new Date()

const time = new Date('2017-03-12');
console.log(time); // Sun Mar 12 2017 08:00:00 GMT+0800 (CST)

const time = new Date('2017-3-12');
console.log(time); // Invalid Date

const time = new Date('2017-03-12 12:00:00');
console.log(time); // invalid Date

（2）.使用moment

const time = moment('2017-03-12').format('YYYYMMDD');
console.log(time); // "20170312"

const time = moment('2017-3-12').format('YYYYMMDD');
console.log(time); // "Invalid date"

const time = moment('2017-03-12 12:00:00').format('YYYYMMDD');
console.log(time); // "20170312"

const time = moment('2017-3-12 12:00:00').format('YYYYMMDD');
console.log(time); // "Invalid date"

3.chrome 全部解析正常

4.IOS 中new Date() 不能解析的原因

在IOS5以上版本（不包含IOS5）中的Safari浏览器能正确解析出Javascript中的 new Date(‘2017-03-12’) 的日期对象。不能解析new Date(‘2017-03-12 12:00:00’)的日期对象

但是在IOS5版本里面的Safari解析new Date(‘2017-03-12’) 就不正确，在IOS5的Safari中返回的永远是”Invalid Date”。

YYYY-MM-DD格式是包含在标准中的，只是Safari没有实现, 解决办法： 使用‘/’ 代替 ‘-‘ new Date(‘2017/03/12 12:00:00’);

5.IOS 中moment() 不能解析的解决办法

使用moment(‘时间’, ‘YYYY-MM’).format(‘YYYYYMM’)代替
