    Laravel API Documentation Generator 编写简单优雅的API文档

    准备条件 如果laravel/framework的版本是5.4或者更高的版本，安装2.0
    如果laravel/framework的版本低于5.4， 安装1.0

    1.安装

    安装命令

    composer require mpociot/laravel-apidoc-generator 1.7.0

    修改 config/app.php 目录， 添加

    Mpociot\ApiDoc\ApiDocGeneratorServiceProvider::class,

    2.可用命令选项

    - output 设置文档的输出路径，默认为: public/docs

    php artisan api:generate --routePrefix="api/v1/pad/*" --output="public/pad"

    在public目录下生成pad文件夹，如下图所示

    - routePrefix 生成文档的路由前缀, 必须提供

    php artisan api:generate --routePrefix="api/v1/pad/*"

    3.使用

    要生成自己的API文档，使用api:generate artisan命令。

    php artisan api:generate --routePrefix="api/v1/*"

    其原理是通过扫描routes.php为指定路由生成相应API文档,这个命令将扫描程序中路由是 api/v1/*

    带参数的API

    如果我们请求某个API时会带参数，如何生成API文档

    • 在定义应控制器方法之前我们先通过如下命令生成一个请求类：

      php artisan make:request TestRequest

    • 这会在app/Http/Requests目录下新生成一个TestRequest类，我们编辑该类的rules方法如下：

      public function rules(){
    return [
        'title' => 'required|max:255',
        'body' => 'required',
        'type' => 'in:foo,bar',
        'thumbnail' => 'required_if:type,foo|image',
    ];
}

    • 接下来再去控制器中定义相应方法：

      public function params(Requests\TestRequest $request){
    // TODO
}
